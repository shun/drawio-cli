# drawio-cli Requirements Document

## 1. プロジェクトの背景と目的
現在、LLM (AIエージェント) を用いたドキュメント作成において、アーキテクチャ図やフローチャートなどの視覚的なダイアグラムの生成は主に Mermaid.js が使用されています。しかし、Mermaid は表現力に限界があり、自由なレイアウトやアイコンの配置、詳細な配色の指定が難しいという課題があります。

一方で、高い表現力を持つ draw.io は、複雑なブラウザ依存の描画エンジンを持っているため、LLMが直接SVGとして出力することが不可能です。

本プロジェクトの目的は、**「LLMが生成した draw.io 用のXML設計図を、正確かつ美しくレンダリングできるローカルCLIツール (`drawio-cli`) を提供すること」** です。これにより、LLM の持つ論理構築力と、draw.io の持つ高度な描画力、さらに人間による直感的な GUI 編集をシームレスに連携させるワークフローを実現します。

## 2. システムスコープ
本ツールは、Node.js 環境で動作する CLI アプリケーションです。
主要な機能として、XMLベースの `.drawio` ファイルから `.svg` ファイルへの変換（エクスポート機能）と、ローカルサーバーを介した `.svg` ファイルのブラウザ編集（エディット機能）を提供します。

## 3. 機能要件 (Functional Requirements)

### 3.1. エクスポート機能 (Export)
* **概要**: 指定された draw.io フォーマットのファイル (XML) を読み込み、描画済みの SVG ファイルを出力する。
* **要件 3.1.1**: 入力ファイルとして `.drawio`, `.xml`, `.svg` (draw.ioメタデータを内包するもの) の拡張子を受け付けること。
* **要件 3.1.2**: ヘッドレスブラウザ (Playwright) とローカルにバンドルされた draw.io アセットを使用して、図形、テキスト、フォント、数式(MathJax)等を本家ツールと同一の品質でレンダリングすること。
* **要件 3.1.3**: 出力される SVG ファイルは、単なる画像として表示可能であると同時に、`content` 属性として元の XML データを完全に内包 (`.drawio.svg` フォーマット) していること。
* **要件 3.1.4**: ユーザーが `-o` または `--output` オプションで出力先のファイルパスを自由に指定できること。指定がない場合は、入力ファイル名の拡張子を `.svg` に置換して出力すること。

### 3.2. 編集機能 (Edit)
* **概要**: 生成済みの `.drawio.svg` ファイルをローカルのブラウザで直接開き、GUI上で編集・上書き保存できるようにする。
* **要件 3.2.1**: 指定された `.drawio.svg` ファイルから XML メタデータを読み取れるか事前に検証し、メタデータが存在しない場合はエラーとすること。
* **要件 3.2.2**: 編集用のローカル Web サーバー (Express) を立ち上げ、OSのデフォルトブラウザを自動的に開くこと。
* **要件 3.2.3**: ブラウザ内の iframe で draw.io エディタ (`index.html?embed=1&modify=1`) をロードし、`postMessage` 通信を用いてローカルサーバーから XML データをエディタに渡すこと。
* **要件 3.2.4**: ユーザーがブラウザ上で「保存」操作を行った際、更新された XML データを受け取り、それを再度エクスポートモジュール経由で SVG としてレンダリングした上で、元のファイルを安全に上書き保存すること。

### 3.3. コアアセットのバンドルと最適化
* **概要**: CLIが外部ネットワークに依存せず動作するよう、公式の draw.io アセットをツールに内包するが、そのサイズを最小限に抑える。
* **要件 3.3.1**: CLI の実行に不要な巨大ファイル（例: `integrate.min.js`, サードパーティ連携用HTML, 大部分の `stencils`）をビルドプロセス (`copy-drawio.js`) で除外すること。
* **要件 3.3.2**: バンドルサイズを 60MB 以下（初期状態 160MB から大幅削減）に保ち、かつ エクスポート機能およびエディタ機能の実行において致命的なエラー（404等によるクラッシュ）が発生しないこと。

### 3.4. LLM用 AgentSkill の提供
* **概要**: LLM が自律的に本ツールを利用できるようにするためのスキル定義（プロンプト・スクリプト群）を提供する。
* **要件 3.4.1**: LLM が draw.io の XML をゼロから記述できるよう、ノード・エッジ・スタイルの基礎をまとめたリファレンスドキュメント (`drawio-xml-basics.md`) を同梱すること。
* **要件 3.4.2**: 最終成果物として `.drawio.svg` を残すワークフローを定義し、人間に対して「VSCode拡張機能などで後からGUI編集可能である」旨を案内するよう LLM に指示すること。

## 4. 非機能要件 (Non-Functional Requirements)

* **実行環境**: Node.js v18 以上で動作すること。
* **依存関係**: ブラウザ操作のために `playwright` に依存するが、可能な限りその他の外部ライブラリへの依存を最小限にすること。
* **パフォーマンス**: エクスポートコマンドは、極端に巨大な図でない限り、数秒以内（ブラウザの起動〜保存完了まで）に処理を完了すること。
* **堅牢性**: 無効なXMLや存在しないファイルパスが渡された場合は、明確なエラーメッセージを標準エラー出力 (stderr) に出力し、プロセスを異常終了 (Exit Code 1 等) させること。

## 5. 前提条件と制約事項
* CLI の実行環境（CI/CD サーバーを含む）には、Playwright が Chromium ブラウザを起動できる環境（必要なOSライブラリ等）が整っていることを前提とする。
* 生成される図の見た目（テキストの折り返しやフォントのレンダリング）は、実行環境の Chromium の仕様やシステムにインストールされているフォントに一部依存する可能性がある。
